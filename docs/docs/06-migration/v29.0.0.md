## Upgrade to v29.0.0

:::tip
This section is being worked on. Please check back later.
:::

For the Ignite v29 Release we have a few changes that we would like to communicate with you.

In this document, we will go through the changes from v28 to v29. If you are looking to migrate from an earlier version, please use our tool "generate migration differences".

[How to use gen-mig-diffs](https://tutorials.ignite.com/guide-to-use-gen-mig-diffs-for/).

Coming from v28 and migrating to v29, here's what changes:

- When scaffolding your blockchain with v29, you now have a `Makefile` that allows you to build or install the blockchain convinient on any system. 
    - Commands available are:
        - Tests: `make test`, `make test-unit`, `make test-race`, `make test-cover`, `make bench`
        - Install: `make all`, `make install`
        - Protobuf: `make proto-deps`, `make proto-gen`
        - Linting: `make lint`, `make lint-fix`
        - Development: `make govet`, `make govulncheck`

- Creating testnets is easier than ever. Use the commands `mychaind in-place-testnet --help` to replicate your chain or `mychaind multi-node --help` to spin up your chain with more than 1 validator node.

- Most chains are `sovereign` chains but Ignite also supports ICS, blockchains that are dependend on another. If you have a sovereign blockchain, please add the second line `validation: sovereign` to your `config.yml` file if it does not exist. If you are building an ICS blockchain, please add `consumer` instead of `sovereign`.

- Cosmos SDK has been upgraded to the latest version `v0.53.0` from the previous `v0.50.x` or lower used in earlier Ignite versions.

For the migration, focus on `go.mod`, `config.yml`, `app/`, `x/`, and `proto/` for key changes, such as Cosmos SDK version bumps (v0.50 to v0.52), IBC-go v10, and new scaffolding features in v29.

Below are the differences between version 28.0.0 and v29.0.0-rc1, the most important files.
